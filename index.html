Protekt Demo

A client side password protection utility.

Using the same password everywhere? Protekt generates unique passwords for each site you visit by hashing your original password with the domain name.

How?

Salt

Your original password never gets on the internet. Instead, it gets merged with the domain name of the website you're attempting to log into to prepare a unique starter string which will then be hashed.

Hash

The salted string is now hashed using standard SHA3-256 hash function which produces a 64 character long cipher that cannot be used to get back the original string.

Rebuild

The hash is now rebuilt by taking 4 character blocks from it and replacing them with a character from a 56 character array. Sort of a string representation with an unusual base 56.

Special Characters

The resulting string is appended with a special character that is picked by modding the decimal representation of the cipher with the length of permitted special characters.
